Wooden Heads : Creating Widgets

Created by Gregory Kelly, last modified on Oct 23, 2021

Due to the number of files required for each widget (a .cpp, .h and .ui), the Application Project Structure is broken into several subfolders for organization. 
In order to use CLion (one of the most powerful IDEs available), the project is configured using the CMake toolchain, instead of the proprietary QMake toolchain.

Unfortunately Qt Creator doesn't play nice with this setup, so the below instructions are required for creating a widget.

1. Create files in CLion

In CLion's file explorer, right-click the project root and select New > Qt UI Class

In the dialog that appears, enter the widget name in PascalCase, and ensure that the only selected target is ${PROJECT_WIDGETS}

The parent class can be changed to extend an existing Qt widget

2. Place files in correct locations

CLion will have created the files in the root folder, however this is not the correct place.

The generated header file (in this example mywidget.h), should be placed in include/widgets. It can also be placed in a subfolder in order to group widgets.

The generated source and UI files (in this example mywidget.cpp and mywidget.ui) should be placed in a new subfolder of src/widgets (in this case src/widgets/mywidget). If the header is placed in a subfolder, the new subfolder should follow the same pattern.

The move operations should be performed in CLion as this will automatically update references in CMakeLists.txt. Ensure "Search for references" is checked so the refactor occurs.

3. Add widgets to Qt Creator

Open a widget, right click in empty space, and select "Promoted Widgets".

In the dialog that opens:

  • Select the widget's base class (in this case QWidget)
  • Enter the widget's class name (in this case MyWidget)
  • Enter the include path for the header (in this case widgets/mywidget.h)
  • Select global include

Click add to register the widget
